# VK Integration Skill — end-to-end demo

## Реалистичный пользовательский prompt

> У нас SaaS «Бизнес Пульс» на Next.js, Vercel и Supabase. Сделай вход через VK ID так, чтобы повторный вход возвращал человека в существующий аккаунт, а не создавал дубль. После входа пользователь должен подключить личную страницу и сообщества по ссылкам `vk.com/club123`, `vk.com/public123`, `vk.com/event123` или короткому имени. Статистику собирай в фоне, сохраняй по периодам и показывай `needs_sync`, пока не было успешного обновления. На Vercel появляется ошибка `access_token was given to another ip address`, поэтому не используй пользовательский токен для аналитики. Секреты не должны попадать в браузер и логи.

## Что демонстрирует проект

- два независимых контура: VK ID для identity и `VK_SERVICE_TOKEN` для аналитики;
- `state` + PKCE + короткоживущая httpOnly cookie (политика callback описана в `src/oauth-policy.js`);
- поиск identity до создания пользователя;
- нормализация числовых ссылок и безопасный resolve короткого имени;
- состояния `needs_sync`, `connected`, `error`;
- upsert метрик по `(user_id, platform, channel_id, period)`;
- структурированные ошибки по этапам без значений токенов;
- SQL-схема с уникальными ограничениями и RLS.

Это безопасный исполняемый пример: VK и база заменены in-memory адаптерами. Для production нужно реализовать те же интерфейсы через актуальные документированные VK endpoint и Supabase, не меняя доменную логику.

## Проверенный путь

`browser callback → identity link → community resolve → VK service-token sync → metrics upsert → UI read`

Официальные страницы VK ID и VK API были запрошены 20.07.2026, но среда проверки не смогла их открыть. Поэтому URL, версия API и параметры намеренно не захардкожены: перед production-подключением их надо сверить в документации для конкретного типа VK-приложения.
